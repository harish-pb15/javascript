## 4 pillars
Abstraction
Encapsulation
Inheritance
Polymorphism

1. Encapsulation 📦

Definition: Bundling data (properties) and methods (functions) together inside a single class/object.

One line: Data + Methods = One Unit

Example: A BankAccount class contains balance and methods like deposit() and withdraw().

2. Abstraction 🎭

Definition: Hiding internal implementation and exposing only the necessary functionality.

One line: Hide complexity, show essentials

Example: Using an ATM without knowing how it processes transactions internally.

3. Inheritance 👨‍👩‍👧

Definition: A class acquires the properties and methods of another class.

One line: Reuse code from a parent class

Example: Dog inherits eat() from Animal.

4. Polymorphism 🔄

Definition: The same method can have different behaviors for different objects.

One line: One interface, many implementations

Example: sound() prints "Woof" for Dog and "Meow" for Cat.
